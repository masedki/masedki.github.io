Le but dans ce notebook est de faire sélection de modèle par ridge, lasso et elasticnet et de comparer les modèle en terme d’erreur de classification sur le jeu de données test. Pour chaque type de pénalité, ridge, lasso et elasticnet, nous allons mettre en place une validation croisée à 5 folds. Nous pouvons nous inspirer du code que nous avons développé dans le notebook de sélection de modèle par régularisation dans le jeu de données du cancer de la prostate en adaptant les paramètres

  • family = “binomial”
  • type.measure=“class”

de la fonction cv.glmnet.

Ici l’erreur de prédiction est le taux de mauvais classement dans la prédiction du jeu de données test.
